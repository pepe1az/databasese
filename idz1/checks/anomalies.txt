Аномалии в orders_raw

1. Аномалия вставки

В orders_raw нет отдельной таблицы товаров. Товар существует только как часть
текстового поля product_names внутри заказа.

Пример:
нельзя добавить новый товар "Веб-камера" с ценой 4500, если по нему ещё не было
заказа. Для этого пришлось бы создавать фиктивную строку заказа.

2. Аномалия обновления

Данные клиента повторяются во многих заказах.

Пример:
клиент client42@example.com может встречаться в нескольких строках orders_raw.
Если он сменил телефон, нужно обновить все строки:

UPDATE orders_raw
SET customer_phone = '+79990000042'
WHERE customer_email = 'client42@example.com';

Если обновить только часть строк, у одного клиента будут разные телефоны.

3. Аномалия удаления

Если удалить все заказы клиента, вместе с заказами исчезнут данные самого
клиента: имя, email, телефон и адрес.

Пример:
DELETE FROM orders_raw
WHERE customer_email = 'client42@example.com';

После этого в базе не останется информации о клиенте, даже если она нужна
отдельно от заказов.
