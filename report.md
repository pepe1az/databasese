# Слайд 2

## Что такое потоковая аналитика

- Batch Processing
- Пакетная обработка данных с задержками от минут до часов
- Streaming
- Непрерывная обработка событий в режиме реального времени

# Слайд 3

## Apache Kafka

- Платформа потоковых событий
- Producers отправляют события
- Consumers получают данные
- Topics и Partitions для масштабирования
- Distributed commit log
- Decoupling компонентов

# Слайд 4

## ClickHouse

- Колоночная база данных
- Оптимизация под аналитические запросы (OLAP)
- Быстрые агрегаты
- Высокая скорость выполнения GROUP BY и агрегатных функций
- Сжатие данных
- Эффективное сжатие колонок для экономии памяти
- Не для транзакций
- OLTP-операции не поддерживаются

# Слайд 8

## Ограничения

- 1
- Нет сложной обработки
- Ограниченные возможности для stateful processing
- 2
- Дубликаты событий
- At-least-once delivery гарантирует доставку, но возможны дубли
- 3
- Нет exactly-once
- Точная семантика не гарантируется
- 4
- Порядок событий
- Гарантируется только в пределах партиции

# Слайд 9

## Когда этого недостаточно

- Kafka → Flink → ClickHouse
- Windowing
- Окна времени для агрегаций
- ClickHouse (хранилище)
- Аналитическая запись и запросы
- Stream Joins
- Соединение потоков в реальном времени
- Flink: Stateful
- Состояние между событиями
- Kafka (источник)
- Высокопроизводительный поток событий
